# Core Domain Models

> 8 pure Pydantic models — built first, no DB imports, the foundation of every other layer

---

# TriageCase

**Role:** Central state object — the canonical workflow record passed through the pipeline. Runtime parsing fields stay on the case, while persistence may be split across the main case row and related telemetry/evidence tables.

| field                | type                      | description                                                                                                           |
| -------------------- | ------------------------- | --------------------------------------------------------------------------------------------------------------------- |
| id                   | UUID                      | Unique case identifier                                                                                                |
| repo                 | str                       | Repository identifier in "owner/repo" form                                                                            |
| issue_number         | int                       | GitHub issue number within the repository                                                                             |
| state                | WorkflowState             | Current workflow state                                                                                                |
| state_history        | list[StateTransition]     | Full transition history for the analysis run; persisted in workflow telemetry rather than as a top-level scalar field |
| retry_count          | int = 0                   | Patch generation retry counter, used to route failures back to drafting or to human review                            |
| issue_metadata       | IssueMetadata             | Issue title, body, labels, author, and related GitHub metadata                                                        |
| issue_type           | str \| None               | Normalized issue classification such as "bug", "feature", or "question"                                               |
| severity             | str \| None               | Normalized severity label such as "critical", "high", "medium", or "low"                                              |
| raw_logs             | list[str]                 | Raw log fragments extracted from the issue body or attached text                                                      |
| stack_traces         | list[StackTrace]          | Parsed stack traces extracted during ingestion and normalization                                                      |
| error_messages       | list[str]                 | Extracted exception names, error codes, or other error signatures                                                     |
| evidence             | EvidencePack \| None      | Retrieval output assembled for reasoning and stored as the case’s grounded evidence                                   |
| candidate_duplicates | list[DuplicateCandidate]  | Vector-similar issues detected during retrieval                                                                       |
| hypotheses           | list[RootCauseHypothesis] | Ranked root-cause hypotheses generated by reasoning                                                                   |
| patch_drafts         | list[PatchDraft]          | Generated patch candidates, usually one per attempt                                                                   |
| validation_result    | ValidationResult \| None  | Validation outcome from the isolated worktree after patch application                                                 |
| pr_draft             | PRDraft \| None           | Draft pull request metadata prepared after successful validation                                                      |

# EvidencePack

**Role:** Grounded evidence object produced by retrieval and used as the sole input to reasoning. This is the retrieved snapshot of context for a triage case.

| field                 | type                     | description                                                                   |
| --------------------- | ------------------------ | ----------------------------------------------------------------------------- |
| relevant_excerpts     | list[str]                | Key issue-body excerpts selected during retrieval                             |
| stack_trace_frames    | list[StackFrame]         | Structured stack frames with file, line, function, and local variable details |
| error_signatures      | list[str]                | Error names, codes, HTTP statuses, and other extracted signatures             |
| file_references       | list[FileReference]      | Repository file slices retrieved for context                                  |
| similar_code_snippets | list[CodeSnippet]        | Relevant code excerpts retrieved from the cloned repository                   |
| duplicate_candidates  | list[DuplicateCandidate] | Candidate duplicate issues ranked by similarity                               |
| related_issues        | list[RelatedIssue]       | Related issues retrieved from the vector store                                |
| retrieval_timestamp   | datetime                 | Timestamp of the retrieval snapshot for caching and replay                    |
| embedding_model       | str                      | Embedding model used to produce retrieval vectors                             |
| confidence            | float                    | Retrieval quality score used by downstream ranking and gating                 |

# RootCauseHypothesis

**Role:** A single ranked hypothesis with evidence citations. The top-ranked hypothesis with confidence ≥ θ drives patch generation.

| field               | type                     | description                                                                                             |
| ------------------- | ------------------------ | ------------------------------------------------------------------------------------------------------- |
| id                  | UUID                     | Unique hypothesis identifier                                                                            |
| rank                | int                      | Rank of the hypothesis, where 1 is the strongest candidate                                              |
| hypothesis          | str                      | Human-readable root-cause explanation                                                                   |
| category            | str                      | Normalized hypothesis category such as "null-ref", "race-condition", "config-error", or "type-mismatch" |
| confidence          | float                    | Calibrated confidence score between 0.0 and 1.0                                                         |
| evidence_references | list[str]                | References to the EvidencePack items that support this hypothesis                                       |
| affected_files      | list[str]                | Repository-relative file paths implicated by the hypothesis                                             |
| affected_lines      | list[tuple[str,int,int]] | File and line ranges implicated by the hypothesis                                                       |
| model_used          | str                      | LLM model used to generate the hypothesis                                                               |

# PatchDraft

**Role:** A generated patch ready to be applied to a git worktree for validation.

| field              | type      | description                                                     |
| ------------------ | --------- | --------------------------------------------------------------- |
| id                 | UUID      | Unique patch identifier                                         |
| hypothesis_id      | UUID      | Hypothesis this patch is intended to address                    |
| diff_content       | str       | Unified diff content generated for worktree application         |
| files_modified     | list[str] | Repository-relative file paths touched by the patch             |
| branch_name        | str       | Temporary branch name used for patch validation and publication |
| commit_message     | str       | Proposed commit message in Conventional Commits style           |
| explanation        | str       | Natural-language explanation of the fix and why it should work  |
| generation_attempt | int       | Patch generation attempt number, starting from 1                |
| model_used         | str       | LLM model used to generate the patch                            |

# ValidationResult

**Role:** Output of running tests, build, and lint on the patch inside an isolated git worktree.

| field            | type         | description                                                                                  |
| ---------------- | ------------ | -------------------------------------------------------------------------------------------- |
| patch_draft_id   | UUID         | Patch draft that was validated                                                               |
| tests_passed     | bool \| None | Whether the test suite passed; None when no test suite is detected                           |
| tests_output     | str \| None  | Captured stdout and stderr from the test command                                             |
| build_passed     | bool \| None | Whether the build step passed; None when no build command is available                       |
| build_output     | str \| None  | Captured stdout and stderr from the build command                                            |
| lint_passed      | bool \| None | Whether linting passed for the modified files                                                |
| lint_output      | str \| None  | Captured stdout and stderr from linting                                                      |
| overall_passed   | bool         | True only when all relevant validation steps pass                                            |
| confidence_delta | float        | Adjustment applied to the top hypothesis confidence after validation                         |
| worktree_path    | str          | Temporary worktree path used during validation; typically treated as ephemeral runtime state |

# DuplicateGroup

**Role**: A vector-detected cluster of triage cases that describe the same underlying bug.

| field             | type             | description                                                            |
| ----------------- | ---------------- | ---------------------------------------------------------------------- |
| id                | UUID             | Group identifier                                                       |
| primary_case_id   | UUID             | Representative triage case for the cluster                             |
| member_case_ids   | list[UUID]       | All triage case identifiers in the cluster, including the primary case |
| similarity_scores | dict[str, float] | Mapping from case ID to similarity score against the primary case      |
| cluster_summary   | str              | LLM-generated summary of the shared root cause                         |
| threshold_used    | float            | Similarity threshold used to form the cluster                          |

# AnalysisRun

**Role:** Persistent record of a complete triage execution — enables evaluation, replay, and debugging.

| field                | type                  | description                                            |
| -------------------- | --------------------- | ------------------------------------------------------ |
| id                   | UUID                  | Run identifier                                         |
| triage_case_id       | UUID                  | TriageCase processed by this run                       |
| model_used           | str                   | LLM model name used for reasoning and patch generation |
| embedding_model      | str                   | Embedding model name used for retrieval                |
| confidence_threshold | float                 | Branching threshold used during the run                |
| duration_seconds     | float                 | Total wall-clock duration of the run                   |
| state_transitions    | list[StateTransition] | Full transition log captured for replay and debugging  |
| workflow_trace       | list[TraceEntry]      | Verbose execution trace of the run                     |
| final_state          | WorkflowState         | Terminal workflow state reached by the run             |

# PRDraft

**Role:** PR metadata assembled before GitHub publication. Created only after validation passes.

| field            | type        | description                                                                                  |
| ---------------- | ----------- | -------------------------------------------------------------------------------------------- |
| id               | UUID        | Draft pull request identifier                                                                |
| patch_draft_id   | UUID        | Patch draft that this PR publishes                                                           |
| title            | str         | Pull request title following the project’s publication format                                |
| body             | str         | Pull request body containing the root cause summary, fix explanation, and validation results |
| base_branch      | str         | Target branch for the PR                                                                     |
| head_branch      | str         | Branch containing the validated patch                                                        |
| labels           | list[str]   | Labels to attach to the pull request                                                         |
| github_pr_number | int \| None | GitHub PR number after successful publication                                                |
| pr_url           | str \| None | Full GitHub pull request URL                                                                 |
| status           | str         | Publication state such as "pending", "published", or "rejected"                              |
